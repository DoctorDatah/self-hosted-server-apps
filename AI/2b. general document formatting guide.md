- **Goal:** Standardize formatting for all repo docs.

- **Last updated:** 2026-01-25

## Purpose
- **Make docs easy to scan** with consistent structure.
- **Keep reviews fast** by using predictable formatting patterns.
- **Reduce rework** by clarifying where key information lives.

## Scope
- **Applies to:** all documentation in this repo.
- **Exceptions:** Docs with a dedicated format guide override this one.

## Scanability Rules
- **Start with a 1-2 line summary** at the top.
- **Use consistent headings** and section order.
- **Prefer single-line bullets** for each idea.
- **Use tables** for variables, decisions, and mappings.

## Heading Structure
- **H1:** Document title.
- **H2:** Major sections.
- **H3:** Subsections only when needed.

## Front Matter (required)
- **Title**
- **Project**
- **Owner**
- **Status**
- **Last updated**
- **Related docs**

## Writing Style
- **One idea per line.**
- **Use concrete nouns and verbs.**
- **Avoid filler phrases and vague qualifiers.**
- **Use active voice.**

## Bullets and Lists
- **Max 4-6 bullets** per section.
- **Keep bullets parallel** in grammar and tense.
- **Use numbered lists** for steps or requirements.
- **Avoid nested bullets**; use tables or labeled bullets instead.

## Tables
- **Use tables** for decisions, variables, and comparisons when relevant.
- **Keep headers short** and descriptive.
- **Keep long explanations** in text, not tables.

## Default Sections (format only)
1) **Summary**
2) **Front Matter**
3) **Purpose**
4) **Scope**
5) **Requirements** (optional)
6) **Implementation** (optional)
7) **Risks and Mitigations** (optional)
8) **Open Questions** (optional)
