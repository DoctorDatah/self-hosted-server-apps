# Cloudflare Tunnel (VM)

Flat, self-contained setup to run Cloudflare Tunnel on a VM via Docker Compose.

## Files
- `docker-compose.yml` - cloudflared service
- `config.yml` - ingress config template (no secrets)
- `config.generated.yml` - generated by `install.sh` when tag selection is used
- `requirements.txt` - pinned cloudflared image tag
- `install.sh` - start the tunnel (prompts for tag selection)
- `logs.sh` - follow logs
- `vm-requirements.txt` - pinned Docker packages for the VM
- `Step by Step Installation \\ Setup Guide/1. repo-clone-guide.md` - clone/update the repo on the VM
- `Step by Step Installation \\ Setup Guide/2. cloudflare-tunnel-creation-guide.md` - create the tunnel in Cloudflare
- `Step by Step Installation \\ Setup Guide/3. cloudflare-install-guide.md` - install Docker (if needed) and start the tunnel
- `Step by Step Installation \\ Setup Guide/4. [In self hosted app config]  cloudflare-to-app-network-guide.md` - app network guidance
- `Step by Step Installation \\ Setup Guide/5. cloudflare-troubleshooting-guide.md` - troubleshooting

## Usage (on the VM)
```bash
export CLOUDFLARE_TUNNEL_TOKEN="your_token_here"
./install.sh
./logs.sh
```

## Notes
- Update `config.yml` with your hostname and service target.
- `install.sh` can filter `config.yml` by tag blocks and write `config.generated.yml`.
- `CLOUDFLARE_TUNNEL_TOKEN` must be exported in the shell session.
